# DevHaven v3.0.0 Release Notes

- 发布日期：2026-03-20
- Tag：`v3.0.0`
- 对比版本：`v2.8.3` -> `v3.0.0`

## 本次升级内容

### 1. Swift 原生主线正式并入主仓库
- 将 `swift` 分支完整合并到 `main`，把 `macos/` 下的原生应用、Ghostty 终端承载、原生 worktree 流程与 Swift 测试基线正式带入主线。
- 原生工作区已经具备“左侧已打开项目 + 右侧标签页 / 分屏终端区”的主壳能力，并继续兼容 `~/.devhaven/*` 数据目录与项目级文档。

### 2. 旧的 React / Tauri 兼容源码已从仓库移除
- 已删除旧的前端与桌面壳源码目录：`src/`、`src-tauri/`、`scripts/`、`public/`。
- 已删除旧的 Node / Vite / pnpm 元数据与构建配置：`package.json`、`package-lock.json`、`pnpm-lock.yaml`、`pnpm-workspace.yaml`、`vite.config.ts`、`unocss.config.ts`、`tsconfig*.json`、`index.html`、`.nvmrc`。
- 自 `v3.0.0` 起，仓库只保留 `macos/` 原生应用主线与相关文档。

### 3. 发布形态切换为 macOS 原生 App
- GitHub Release 自 `v3.0.0` 起只构建并上传 **macOS 原生 `.app` 压缩包**。
- 发布工作流已从 `tauri-action` 切换为 **纯 Swift/macOS 原生链路**：`swift test --package-path macos` + `bash macos/scripts/build-native-app.sh`，不再依赖 Node / pnpm。

### 4. 原生打包元数据已收口到 `macos/`
- 新增 `macos/Resources/AppMetadata.json` 作为原生打包真相源，持有 `productName`、`bundleIdentifier`、`version`。
- `macos/scripts/build-native-app.sh` 已改为读取 `macos/Resources/AppMetadata.json` 与 `macos/Resources/DevHaven.icns`，不再依赖任何旧的 Node / Tauri 配置文件。

## 版本升级

本次发布将原生版本统一升级为 `3.0.0`：
- `macos/Resources/AppMetadata.json`

## 兼容性说明

- 本地数据目录仍为 `~/.devhaven/`，无新增迁移步骤。
- 本地开发仍需要准备 `macos/Vendor/` 的 Ghostty framework / resources；该目录由 `.gitignore` 忽略，不会入库。
